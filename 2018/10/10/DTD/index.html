DTD语法：

  • DTD中的元素：

    • 元素定义语法

    • 元素类型：

      | 元素类型 | 含义 |
      | :— | :—– |
      | EMPTY | 空元素，即该元素不包括任何内容（建议写成单标签形式） |
      | (#PCDATA) | 字符串类型，即该元素内容可以为任意字符串 |
      | ANY | 任意类型，内容无限制 |
      | 子元素 | 子元素类型，该元素嵌套子元素 |
      | 混合类型 | 混合类型，该元素内部既包含字符串又包含子元素 |

    • 子元素：

      • 使用()将所有子元素括起来，当子元素之间使用(,)分隔时表示子元素是有序的；
      • 当子元素之间使用(|)分隔时，表示子元素互斥；

      • 使用修饰符时为无序组合子元素：

        |修饰符|含义|
        | :— | :— |
        | + | 一次或多次 |
        | ? | 零次或1次 |
        | * | 零次或多次 |
        | 省略 | 一次 |

  • DTD中的属性：

    • 属性定义语法：

      • 约束：

        | 修饰符 | 含义 |
        | :– | :– |
        | #REQUIRED | 该属性必须 |
        | #IMPLIED | 该属性可有可无 |
        | #FIXED | 表示XML中只会给出元素定义时设置的固定默认值，同时只有当约束为#FIXED时才能使用默认值 |

    • 属性类型：

      | 属性类型 | 说明 |
      | :— | :——— |
      | CDATA | 单纯字符数据，大部分属性都设置为该值 |
      | ID | 具有唯一性的属性值，需要注意，该属性值必须以字母开头 |
      | IDREF | 引用其他属性ID的值，该值必须在其他ID属性中存在 |
      | IDREFS | 引用多个属性ID的值，用空格分隔 |
      | ENTITY | 未解析的外部实体类型 |
      | ENTITYS | 多个未解析的外部实体，空格分开 |
      | NMTOKEN | 即Name Token，就是关键字的名字，可以包含字母、数字、.、——: |
      | NMTOKENS | ~ |
      |NOTATION | 标记名称 |
      |Enumerated | 枚举类型的属性，只能从已经有的属性中选取，不能输入新的项目 |

    • DTD中的实体与C语言中的宏定义、Java中的常亮定义类似。预先定义一个实体，其可以包含一些常用的文字内容或者二进制内容。
