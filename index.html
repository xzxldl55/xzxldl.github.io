XML之命名空间

发表于 2018-10-29

背景：

由于XML中的元素名称是由开发者定义的，当两个开发文档使用相同的元素名时就会发生冲突。此时则使用前缀来避免冲突

<s:table>桌子</s:table>

此处s即为其命名空间。

简述：

  • 定义：
    • name：即空间别名，可自定义，不写则为默认命名空间
    • 命名空间字符串：为一个URI，但是可以不需要指向任意内容，可以是一个假的，只要保证其唯一性就可以

      xmlns[:name] = “命名空间字符串” //别名不能包含:与关键字

搜索引擎与信息检索的基础知识

  • 参考书目：
    • 《Lucene搜索引擎开发进阶实战》-成龙
    • 《搜索引擎（信息检索实践）》刘挺等
  • 1.搜索引擎-概念：
    • 是一种在web上应用的软件系统。其以一定的策略在web上搜集与发现信息。在进行一定处理后，为用户提供web信息查询服务。
    • 采集->处理->建立索引
    • 澄清问题：
      • 首先搜索引擎的搜索实在系统内部进行的，而非在网络间搜索的。
      • 其次，用户访问的网页是原始出处，而非系统内部信息资源。
      • 导致：结果列表上的标题与摘要内容与点击URL看到的内容不一样。（其系统内部爬虫周期性爬取信息，与真实网络信息有出入）
  • 2.发展史
    • 第一代：分类目录时代
      • 导航网站，也可以说分类目录网站。（YaHoo）
    • 第二代：文本检索时代：
      • 用户输入查询信息，提交至服务器进行检索，服务器在将检索结果反馈给用户。
      • 其信息检索模型主要包括：布尔模型，向量空间模型
    • 第三代：整合分析时代
      • 将用户输入关键字，反馈回来的海量信息，智能整合成一个门户网站式的界面，而非向第二代一样直接返回一个清单。（goole）
    • 第四代：用户中心时代：
      • 以用户为中心，当用户输入查询请求时，同一个查询请求的关键词在用户的背后可能是不同的查询要求。（eg：搜索“苹果”–>iphone || apple）（淘宝）
    • 第五代：生活生态圈搜索时代
      • 基于物联网的搜索，比如找东西，远程看管小孩等等。
  • 3.搜索引擎分类：
    • 全文搜索引擎：如goole、百度，他们从互联网提取各个网站的信息，建立数据库，并能检索与用户查询条件相匹配的记录，并返回结果。
    • 目录搜索引擎：如YaHoo、新浪。用户按照分类目录找到所需要的信息，不依靠关键词进行查询。
    • 元搜索引擎：如InfoSpace、Dogpile、Vivisimo在接受用户查询请求后，同时在多个搜索引擎上搜索，并将结果返回给用户。（元数据：数据的数据–>元搜索引擎：搜索引擎的搜索引擎）==>问题：效率低
    • 分布式搜索引擎：按区域、主题或者其他标准创建分布式索引服务器。

  • 4.搜索引擎原理：

    • 1.搜集器:
      • 定期搜集：每次搜集替换上一次的内容。每次都是从新来一次。
      • 增量搜集：开始时搜集一批，往后只是搜集新出现的、有改变的、不存在了的删除。（广度、深度优先）
    • 2.索引器：
      • 倒排文档：使用文档中所含关键词为索引，文档作为索引目标的一种结构。
      • 关键词提取：从网页源文件中提取出能够代表它内容的一些特征（分词…权重计算）使用词袋或者关键词向量来表示网页。
    • 3.查询器：
      • 查询方式：指系统允许用户提交查询的形式。（简单检索、高级检索、专家检索）
      • 匹配：匹配过程是用户提交的检索式与倒排文件进行相似度计算和求交集的过程。
      • 结果排序：按照某种评价方式，确定出检索结果集合中元素相关性程度的过程。（eg：pageRanke）
      • 文档摘要：
        • 静态生成：独立于查询，按照某种规则，事先在预处理阶段从网页内容提取出一些文字，如截取网页正文的开头512字节；或者将每个段落的第一句拼接起来。
        • 动态生成：在响应查询时，根据查询词在文档中的位置提取出周围的文字来，在显示时将查询词标亮。
    • 4.用户接口：
      • 用户接口UI：搜索框
      • 复杂接口可以让用户对查询进行限制。
    • 5.搜索引擎的评价：
      • 信息检索效率
        • 查全率
        • 查准率
        • F值

DTD语法

发表于 2018-10-10

DTD语法：

  • DTD中的元素：

    • 元素定义语法

    • 元素类型：

      | 元素类型 | 含义 |
      | :— | :—– |
      | EMPTY | 空元素，即该元素不包括任何内容（建议写成单标签形式） |
      | (#PCDATA) | 字符串类型，即该元素内容可以为任意字符串 |
      | ANY | 任意类型，内容无限制 |
      | 子元素 | 子元素类型，该元素嵌套子元素 |
      | 混合类型 | 混合类型，该元素内部既包含字符串又包含子元素 |

    • 子元素：

      • 使用()将所有子元素括起来，当子元素之间使用(,)分隔时表示子元素是有序的；
      • 当子元素之间使用(|)分隔时，表示子元素互斥；

      • 使用修饰符时为无序组合子元素：

        |修饰符|含义|
        | :— | :— |
        | + | 一次或多次 |
        | ? | 零次或1次 |
        | * | 零次或多次 |
        | 省略 | 一次 |

  • DTD中的属性：

    • 属性定义语法：

      • 约束：

        | 修饰符 | 含义 |
        | :– | :– |
        | #REQUIRED | 该属性必须 |
        | #IMPLIED | 该属性可有可无 |
        | #FIXED | 表示XML中只会给出元素定义时设置的固定默认值，同时只有当约束为#FIXED时才能使用默认值 |

    • 属性类型：

      | 属性类型 | 说明 |
      | :— | :——— |
      | CDATA | 单纯字符数据，大部分属性都设置为该值 |
      | ID | 具有唯一性的属性值，需要注意，该属性值必须以字母开头 |
      | IDREF | 引用其他属性ID的值，该值必须在其他ID属性中存在 |
      | IDREFS | 引用多个属性ID的值，用空格分隔 |
      | ENTITY | 未解析的外部实体类型 |
      | ENTITYS | 多个未解析的外部实体，空格分开 |
      | NMTOKEN | 即Name Token，就是关键字的名字，可以包含字母、数字、.、——: |
      | NMTOKENS | ~ |
      |NOTATION | 标记名称 |
      |Enumerated | 枚举类型的属性，只能从已经有的属性中选取，不能输入新的项目 |
